**What It Does**
- Entry point for interview prep docs covering both frontend and backend.
- Explains how the documentation mirrors the actual repo structure.
- Notes which runtime/generated files are intentionally excluded.

**Why This Structure**
- Keeps study material centralized and easy to navigate.
- Mirrors the codebase so you can jump between docs and code quickly.
- Excludes local runtime artifacts (`data/`, `.env`, `node_modules/`, `dist/`) to keep focus on source.

**Principles**
- Documentation as an architectural map, not just a tutorial.
- Consistency and traceability from docs to source files.
